## Implementation Hints

### 1. Kokoro Announcer
- Run Kokoro offline after script gen, before Bark loads
- Render Announcer lines to 24kHz WAV, resample to 48kHz with GPU torchaudio
- Keep as discrete track, crossfade 0.1s linear with Bark scene audio in EpisodeAssembler
- Do NOT attempt to blend — route as separate bus

### 2. pedalboard Announcer chain
```python
from pedalboard import Pedalboard, TapeSaturation, Compressor
announcer_chain = Pedalboard([
    TapeSaturation(saturation=0.3, high_boost=0.1),
    Compressor(threshold_db=-18, ratio=3, attack_ms=5, release_ms=50),
])
```

### 2b. Scene bus harmonic exciter
- Synthesize upper harmonics from Bark's 24kHz-limited output
- Apply after scene crossfade, before master bus
- Fixes timbre mismatch between Kokoro (clean) and Bark (muffled)

### 3. ffmpeg video overhaul
```
-filter_complex "[a:0]avectorscope=s=1920x1080:zoom=1.3:rc=0:gc=200:bc=0[v];
[v]scanlines=strength=0.2,grain=size=2,vignette=angle=0.3[out]"
```
- Glowing green CRT radar reacting to dialogue = SIGNAL LOST aesthetic
- Optional: SVG radio dial overlay in corner via overlay filter

### 4. pyloudnorm -16 LUFS master
```python
import pyloudnorm as pyln
meter = pyln.Meter(48000)
loudness = meter.integrated_loudness(final_audio)
final_audio = pyln.normalize.loudness(final_audio, loudness, -16.0)
```

### 5. Intro/outro sting
- 5-second A-minor arpeggio + rising static noise sweep
- scipy.signal.chirp + numpy, fully deterministic
- Prepend and append in EpisodeAssembler

### 6. Parallel pipeline overlap
- While Gemma writes script to disk, queue Bark TTS for previous scene on CPU
- multiprocessing.Queue between stages
- Target ~30% wall-time reduction

### 7. MP4 chapter markers
- EpisodeAssembler already knows scene offsets from crossfade math
- Write ffmetadata text file with CHAPTER sections per `=== SCENE X ===`
- Mux via `-i metadata.txt -map_metadata 1` at encode

## Punted to v1.3

- **Stable Audio Open for SFX** — VRAM fragmentation risk during Gemma↔Bark handoff
- **RVC voice-locking** — 1.5GB extra VRAM + voice model asset management overhead
- **XTTS-v2 / F5-TTS / StyleTTS 2 swap** — marginal quality gain over Bark for the cost
- **Sentiment-driven LUT per scene** — defer until after base video overhaul lands
- **Whisper-tiny subtitle generation** — nice-to-have, not priority
- **Particle fields / Three.js visualizers** — VRAM-hungry for what you get
