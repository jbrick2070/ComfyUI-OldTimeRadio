# Standing Rules for AI Assistants on This Project

Jeffrey Brick. ComfyUI custom node work. Read this before touching anything.

## Code & Story Quality
- Clean code, clean logs, clean names. The reader and the end user matter.
- Every story generated by this pipeline has a start, a middle, and an end. Arcs are the whole point.
- Safe for work. Non-violent. No curse words anywhere — not in code, not in comments, not in sample output.
- Meaningful names. If the name does not tell the reader what the thing does, rename it.

## Regression Discipline
- Always regression test for widget errors and bugs after any change to `INPUT_TYPES`, `widgets_values`, or workflow JSONs.
- Follow the ComfyUI Custom Node Survival Guide Bug Bible:
  https://github.com/jbrick2070/comfyui-custom-node-survival-guide/blob/main/BUG_BIBLE.yaml
- Before declaring done: AST parse the changed files, grep for dead references, verify cross-imports resolve, audit widget count vs `widgets_values` length.

## Git Push Protocol
Git pushes from the AI side are unreliable. Do not try to push twice. One attempt at most — then hand Jeffrey a PowerShell block he can paste.

Every PowerShell block handed to the user MUST:
1. Start with `cd <absolute path>` — never assume the user is in the right directory.
2. Use plain ASCII quotes and plain ASCII hyphens in commit messages. No em-dashes, no smart quotes. PowerShell mangles UTF-8 in command-line arguments.
3. Be copy-paste ready. No placeholders.

Example shape:
```powershell
cd C:\Users\jeffr\Documents\ComfyUI\custom_nodes\ComfyUI-OldTimeRadio
git add <files by name, not "add .">
git commit -m "short ASCII-safe message"
git push origin main
```

## Lockstep Verification (MANDATORY after every push)
After a push, do not trust "it worked." Verify in lockstep:
1. Check GitHub HEAD against local HEAD. They must match exactly.
2. Scan for 0-byte files on the remote.
3. Scan for BOM issues (`head -c3 file | xxd` should show no `EF BB BF`).
4. Scan for truncation (AST parse every .py).
5. Scan for missing node registrations (grep `NODE_CLASS_MAPPINGS`).
6. If any check fails, fix and re-push. Wait and re-check until confirmed clean.

This matters because sometimes files get into weird states on disk — encoding corruption, stale caches, phantom diffs. Verify with `Get-Content | Select-String` before declaring a change done.

## Encoding on Windows
- UTF-8, no BOM. Always.
- Never use `Set-Content` or `Out-File` for Python source — they inject BOM or mojibake.
- Use `[System.IO.File]::WriteAllText($path, $content, [System.Text.Encoding]::UTF8)` or write from the sandbox side.

## Platform Reality
- Hardware: RTX 5080 Laptop, 16 GB VRAM, Blackwell sm_120.
- Stack: Windows, Python 3.12, torch 2.10.0, CUDA 13.0.
- Attention: SageAttention + SDPA active.
- **Flash Attention 2 is NOT available on this platform. No prebuilt wheel exists for torch 2.10 + CUDA 13 + sm_120 on Windows. Do not attempt to install it. The next AI session should not chase it either — this is documented in the startup log for a reason.**

## Scope Discipline
- 100% local, open source, offline-first.
- No cloud services, no API keys, no paid services.
- VRAM ceiling: 16 GB hardware, 14.5 GB real-world target.
- Sequential execution only. Do not suggest async CUDA streams or queue refactors.

## Autonomy
If a senior pair-programmer would just do the thing, just do it. Install packages, fix errors, sync files, run regression, batch related changes, report a summary at the end. Only escalate when a decision is required or a wall is hit (auth prompts, destructive operations, etc.).
