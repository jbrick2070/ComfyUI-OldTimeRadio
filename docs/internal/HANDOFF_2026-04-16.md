## TL;DR

Phase A of the HyWorld v2.0 next-phase plan is shipped (`5e795a0` + Windows race fix `ed4c44f`), and Phase B v0 is now scaffolded behind a feature flag (`c46a013`). Sidecar contract is race-free (atomic writes with retry on Windows), worker crashes surface as `WORKER_DEAD` instead of hanging the poll loop, malformed `script_json` short-circuits to a `PARSE_ERROR` fallback. SD 1.5 anchor generator is in place but is **default OFF** — `worker.py` only calls it when `OTR_HYWORLD_ANCHOR=sd15` is set in the environment. With the flag unset, behavior is byte-identical to the prior commit. 27 new unit tests cover prompt composition, deterministic seed derivation, cache-key behavior, lazy loader semantics, and per-shot error isolation; all run without torch installed thanks to injectable `infer` and `model_loader` seams. Used today for the Phase B model pick (see below). **Heads-up:** Gemini free-tier daily quota was burned on first call (HTTP 429), so today's round-robin ran half-blind — Claude synthesized off ChatGPT alone. Quota resets daily; bill the API (~$5) if you want the second-opinion safety net always available.

VRAM-coordination work is **deliberately scoped tight**: lightweight scaffold only, no further expansion. Per Jeffrey's reinforced rule (memory file `feedback_no_vram_dragons.md`), the next build cycle goes to **visual quality (Phase B anchors)**, not VRAM safety nets.

**Phase B model picked: SD 1.5 (already on disk), single-pass, no LoRAs.** Trust-but-verify pivot off ChatGPT's SDXL recommendation after a disk audit showed no SDXL weights present and SD 1.5's "muddy broadcast artifact" failure mode plausibly being a *feature* for the SIGNAL LOST aesthetic. ## What shipped in `ed4c44f` (on top of `5e795a0`)

| File | Change |
|------|--------|
| `otr_v2/hyworld/_atomic.py` | New `_replace_with_retry` helper. On Windows, retries `os.replace` up to 8 times with 10 ms × attempt backoff (~80 ms worst case) when it hits `PermissionError`. POSIX path unchanged. `atomic_write_text` now calls the helper. |
| `tests/test_hyworld_phase_a.py` | New `test_concurrent_writers_never_crash` — 3 threads × 100 writes hammer; asserts no writer thread raises and the final file parses as JSON. Also widened reader-side test except clause from `FileNotFoundError` to `(FileNotFoundError, PermissionError)` to mirror what production `poll.py` already catches via the `OSError` parent class. |

**Why this commit existed:** the original Phase A hammer test passed its assertions but writer threads were silently raising `PermissionError [WinError 5]` on Windows because two threads can race the brief exclusive lock that `MoveFileExW` (under `os.replace`) holds. In production this would have manifested as silent dropped `STATUS.json` updates — exactly the kind of bug the atomic-write contract was supposed to prevent. Surfaced by running the suite on Windows after Phase A landed.
