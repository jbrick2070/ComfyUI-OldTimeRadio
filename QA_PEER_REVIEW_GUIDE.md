# ComfyUI-OldTimeRadio Peer Review & QA Guide (v1.4+)

This guide provides the core QA protocols for peer reviewers evaluating pull requests (PRs) or feature branches for the `ComfyUI-OldTimeRadio` custom node suite.

## 1. VRAM & Sequential Offloading Validation
Because OTR runs up to four heavy AI models sequentially (LLM, Bark/Kokoro, MusicGen, and Video Encoders) on single-GPU hardware, reviewers MUST verify memory handling:
*   **Sequential Offloading**: Ensure no two models are loaded into VRAM simultaneously. 
*   **Garbage Collection**: Check that `_unload_llm()` or `force_vram_offload()` are called during node transitions. Code must NOT rely solely on ComfyUI's standard memory manager, as overlapping lazy-loads will blow the 16GB VRAM limit.
*   **Verification Check**: Ask the developer to provide a log showing the `VRAM freed (gc.collect + empty_cache)` outputs during the handoffs.

## 2. LLM Hallucination & Script Parser Fault Tolerance
We have strict safety catch-mechanisms to prevent "Silent Audio Failures" (i.e. generating hundreds of frames of empty video). Reviewers must ensure these safeguards stay intact:
*   **Parser Variant Support**: The `_parse_script()` function handles canonical Gemma output, Mistral Nemo style (next-line dialogue), and shorthand tag variants (`[CHAR, traits]`). If adding more parsing regexes, verify they do not shadow or break existing ones.
*   **The 0-Dialogue Hard Abort (BUG-010)**: If the LLM produces garbage (e.g. just `[MUSIC: Closing theme]` a ~22-char hallucination), `story_orchestrator.py` must reliably throw a `ValueError: Script parser produced 0 dialogue lines`. Reviewers should reject PRs that suppress this error, because allowing an empty script to pass throws downstream video encoders into a loop of rendering blank frames.

## 3. Workflow JSON Integrity
When reviewing newly added `.json` workflows for the frontend:
*   **Widget Blocks**: Ensure string inputs (like `script_text` or `production_plan`) have explicit `{"widget": "string"}` mapping blocks in the JSON. If missing, ComfyUI will drop link connections dynamically, causing defaults (empty strings `[]`) to be passed to nodes.
*   **Cross-Compatibility**: Open the JSON file and verify that nodes aren't referencing local hardcoded system paths `/Users/...` or similar.

## 4. Video & Audio Engine Sync
*   **Audio Assembly**: `EpisodeAssembler` handles normalization and cross-fading. Changes to DSP here should be QA'd for peak clipping.
*   **Video Frame Math**: The total frame count generated by `video_engine.py` MUST match: `int(math.ceil(duration * fps))`. Reviewers checking video duration anomalies should verify that `duration` comes from the `SceneSequencer` audio duration, not a hard-coded length.

## 5. Required Test Regimens for PR Approval
A PR merging into `main` requires a passing run on:
-   **True 16GB Environment**: Running with `mistralai/Mistral-Nemo-Instruct-2407 (quantized=True)` + `suno/bark` + `MusicGen-Medium` without OOM.
-   **Obsidian 4GB Environment**: Running with Kokoro enabled to ensure low-VRAM graceful fallback pathways don't crash.

## Peer Review Checklist:
- [ ] Are LLM context windows being bounded? (`max_new_tokens` limits respected).
- [ ] Does memory flush manually (`gc.collect`) before heavy generation loops?
- [ ] Are edge cases where Bark fails to generate audio handled seamlessly?
- [ ] Are JSON nodes structurally valid and connectable without manual re-wiring?
